**Final Decision: ENGINE + ADAPTER (shared OpenBoxes) + BUILD (sensor
telemetry ingestion)**

OpenBoxes covers the inventory/storage-location/alert layer natively.
For the IoT sensor-telemetry layer specifically, no credible open-source
platform survived the program's personal/student-project screening —
correctly reported as BUILD (per-vendor sensor adapter, Module 4
pattern) rather than adopting an unvetted hobbyist repository.

**Module 18 summary:** OpenBoxes (Eclipse Public License 1.0) resolves 4
of 5 Features as a single Cross-Feature Dependency, the cleanest-
licensed whole-system Engine adoption in the program, and **supersedes
Module 17's tentative `spare-parts-tracking` decision** (see
`stock-management/09-integration-options.md`).
