Atlas CMMS: Docker-based self-hosted deployment, separate backend API
(port 8080) and frontend (port 3000) services, optional MinIO for
object/document storage — a clean microservice-adjacent architecture
that integrates as an independent Engine without absorbing platform
internals, unlike the CMS-coupled SENAITE (Module 14). This is not
Core-Domain-adjacent (asset/maintenance tracking is genuinely a
commodity capability, unlike Laboratory Execution), so wholesale ENGINE
+ ADAPTER adoption carries none of the Core Domain risk flagged
repeatedly in Modules 9-16.
