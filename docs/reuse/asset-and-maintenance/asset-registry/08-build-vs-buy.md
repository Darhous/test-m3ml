Unlike every Feature in Modules 9-16, this Feature is **not** Core
Domain — asset/equipment tracking is a commodity capability with no
ADR-0011 conflict. A genuinely mature, actively-maintained, self-hostable
open-source CMMS (Atlas CMMS) directly matches the Feature's need with
minimal customization risk, making this the program's clearest
ENGINE + ADAPTER opportunity since Module 7 (Apache Superset). Building
a bespoke asset registry would duplicate a solved problem for no
architectural benefit.
