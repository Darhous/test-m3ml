# Candidate Repositories — Message Broker / Queueing

| Candidate | License | Fit |
|---|---|---|
| RabbitMQ (`rabbitmq/rabbitmq-server`) | MPL 2.0 | Durable, moderate-scale, flexible routing — best fit |
| Apache Kafka | Apache-2.0 | Event-streaming/analytics scale — no measured need yet (Constitution Section 55) |
| NATS + JetStream | Apache-2.0 | Sub-millisecond latency, but durability layer (JetStream) is newer/less proven than RabbitMQ's persistence model |
