# Community Review — Message Broker / Queueing

RabbitMQ: mature (est. 2007), broad enterprise adoption, large community,
now Broadcom-stewarded (VMware/Pivotal lineage) — real vendor-continuity
signal, though not foundation-governed. Kafka: Apache Software
Foundation-governed, very large community, but scale-profile mismatch
noted above. NATS: CNCF-hosted, active, smaller broker-specific mindshare
than RabbitMQ/Kafka for classic queueing use cases.
