# License Review — HL7 Integration Engine

| Candidate | License | Compatibility |
|---|---|---|
| Mirth Connect 4.5.2 | MPL 2.0 | **Clear**, but frozen at this version — no security patches beyond it without paying for 4.6+ |
| Apache Camel | Apache-2.0 | **Clear** |
| HAPI HL7v2 | Apache-2.0 | **Clear** |

The real finding is not license text but **license/support trajectory**:
adopting Mirth 4.5.2 means committing to an unpatched engine unless the
platform later pays for NextGen's commercial 4.6+, a `Requires Legal/
Commercial Verification` decision point for a future budget owner, not
this program.
